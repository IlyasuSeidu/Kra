# Customer Support Workflows

## Entry Points
- In-app support center.
- FAQs.
- Live chat.
- Email support.
- AI-assisted chatbot.

## Core Support Jobs
- Explain delivery status.
- Resolve payment questions.
- Handle delay complaints.
- Start lost or damaged package investigation.
- Process refund requests within policy.

## Workflow Rule
Support should be delivery-centric. Every significant support interaction should tie back to a delivery or payment record when possible.

## Approved Launch Channels And Hours
- in-app support for senders
- delivery-linked support entry for receivers
- WhatsApp support line
- phone escalation for urgent `P1`
- support hours: `07:00-19:00` daily

## Ownership
- support inbox owner: `support_admin`
- payment and refund review owner: `finance_admin`
- lost package and operational issue owner: `ops_admin`

## Standard Macros
- status clarification
- payment pending explanation
- refund under review
- failed delivery attempt next steps
- pickup ready instructions

## Closure Rule
- issue may close only when:
  - resolution note exists
  - linked delivery or payment outcome is clear
  - sender-facing response has been sent where applicable
