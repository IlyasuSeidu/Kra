# Security Policy

## Reporting
Report suspected security issues privately to the designated technical and business owners. Do not disclose proof-of-concept exploits publicly before review.

## High-Risk Areas
- auth and session handling
- permissions and admin overrides
- payment provider callbacks
- proof assets and personal data
- offline action replay

## Patch Priority
- Tier 0 correctness or security failures are release-blocking
- auth, payment, and privilege escalation flaws require immediate remediation
