Design Handoff Notes — Kra Sender Dashboard

Colour Tokens

--blue: #2563EB — primary CTA, in-transit status, trusted state
--green: #16A34A — delivered, success, wallet balance, paid status
--amber: #D97706 — payment pending, attention, warning states
--red: #DC2626 — error, issue, failure, report actions
--t1/#111827 primary text · --t2/#374151 secondary · --t3/#6B7280 meta/labels · --t4/#9CA3AF placeholder
--border: #E5E7EB card border · --surf2: #F3F4F6 chip/cell bg

Typography (SF Pro / system-ui)

Greeting: 23px / 700 / tracking −0.3px
Section title: 15px / 700
Card reference: 13px / 700
Card route / body: 12px / 400
Card event/meta: 11px / 400, color t3
Stat number large: 24px / 700 / tracking −0.4px
Badge: 10px / 600 / tracking +0.1px — always pill (border-radius 100px)
Tab label: 9px / 600

Spacing & Radius

Screen horizontal padding: 18px
Card internal padding: 13px sides / 13px top-bottom
Card border radius: 12px · Badge radius: 100px (pill) · Input: 8px
Card gap (stack): 9px
Section spacing (top): 18px
Status stripe: 3px left border, same hue as badge
Shadow: 0 1px 3px rgba(0,0,0,.08), 0 2px 8px rgba(0,0,0,.05)

Motion

Sheet open/close: translateY, 320ms cubic-bezier(0.4,0,0.2,1)
Card press: scale(0.97), 120ms ease
Banner dismiss: translateX(110%) + opacity 0, 300ms ease
Tab switch: instant content swap (no cross-fade in prototype, fade recommended in implementation)
Toast: fade in/out 200ms, auto-dismiss 2.2s
Payment flow: 1.6s spinner → green success → 2.5s auto-close

Screens

Home: greeting, search, 3-stat strip, attention banner, 4 delivery cards, quick actions (2×2), wallet strip, recent shipments
Create Delivery: step 1 of 5 — route selection with origin/destination pickers + price estimate
Track: focused search + active delivery list with status dots
History: filter chips (All/Active/Delivered/Issues) + full 7-delivery list with amounts and badges
Wallet: hero balance card (GHS 340), pending bar, 5 transaction rows
Support: 3 action cards (Track issue / Report problem / Call) + open issues status

Bottom Sheets

Delivery Detail: ref, badge, route, 3-cell grid (service/price/payment), receiver row, timeline, share + report actions. DEL-0043 shows amber payment banner inside.
Payment (MTN MoMo): yellow MTN circle, amount, breakdown, pre-filled phone, spinner → success → auto-close
Top Up: amount chips (GHS 50/100/200/Custom), MoMo field, proceed CTA
Notifications: 2 unread rows (blue dot), tapping payment notif opens payment sheet
Receipt: paid confirmation, delivery details, Download Receipt + Repeat Booking CTAs
Route pickers (From / To): station list with name, city, station ID

Key Business Rules Surfaced in UI

DEL-2026-0043: payment_pending state blocks dispatch — amber banner + card CTA + payment sheet enforce this
MTN MoMo is v1 primary payment path — Hubtel/Paystack deferred
Attention banner persists until user pays or explicitly dismisses (✕)
Receivers don't need accounts — sender shares tracking link
Delivered status shows full doorstep proof timestamp (Apr 25, 3:14 PM)
Launch corridors: Accra–Kumasi (GHS 35), Accra–Tamale (GHS 65), Kumasi–Tamale (GHS 50) base fees shown in route picker
